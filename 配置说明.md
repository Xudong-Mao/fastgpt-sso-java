# FastGPT 泛微 CAS 对接配置说明

本文档用于说明当前 `fastgpt-sso-java` 项目如何按照《ecology统一认证中心V5》中的 CAS 接入方式完成配置，并与 FastGPT 管理端联调。

## 1. 对接原理

当前项目采用泛微文档中推荐的 `weaversso.jar` 接入模式：

1. FastGPT 调用本服务的 `GET /login/oauth/getAuthURL`
2. 本服务返回泛微 CAS 登录地址
3. 浏览器跳转到泛微统一认证中心登录
4. 泛微 CAS 登录成功后回调本服务 `GET /login/oauth/callback`
5. `weaversso.jar` 的 Filter 完成 Ticket 校验，并将 `Assertion` 写入 `HttpSession`
6. 业务代码从 Session 中读取 `AbstractCasFilter.CONST_CAS_ASSERTION`
7. 本服务生成临时 `code` 并重定向回 FastGPT
8. FastGPT 再调用 `GET /login/oauth/getUserInfo?code=xxx` 获取用户资料

项目中的固定取值代码与泛微文档一致，位于：

- [WeaverSsoProviderServiceImpl.java](/Users/xiaohei666/Documents/FastGPT-dev/fastgpt-sso-java/src/main/java/org/example/service/impl/WeaverSsoProviderServiceImpl.java:87)

对应逻辑等价于：

```java
Assertion assertion = (Assertion) request.getSession()
        .getAttribute(AbstractCasFilter.CONST_CAS_ASSERTION);
AttributePrincipal principal = assertion.getPrincipal();
Map<String, Object> attributes = principal.getAttributes();
String ssoLoginName = principal.getName();
```

```xml
<filter>

        <param-name>serverName</param-name>

    </init-param>

<filter>
    <filter-name>SSO Validation Filter</filter-name>
    <filter-class>com.weaver.sso.client.validation.SSO20ProxyReceivingTicketValidationFilter</filter-class>
    <init-param>
        <param-name>ssoServerUrlPrefix</param-name>
        <param-value>http://cas.sc.pbc.gov/sso</param-value>
    </init-param>
    <init-param>
        <param-name>serverName</param-name>
        <param-value>http://应用服务器地址:端口</param-value>
    </init-param>

```

在当前项目中的等价关系如下：

| 文档参数 | 当前项目配置项 | 示例值 |
| --- | --- | --- |
| `ssoServerLoginUrl` | `WEAVER_SSO_SERVER_URL` + `WEAVER_SSO_LOGIN_PATH` + `?appid=` + `WEAVER_SSO_APP_ID` | `http://cas.sc.pbc.gov/sso/login?appid=xxx` |
| `ssoServerUrlPrefix` | `WEAVER_SSO_SERVER_URL + /sso` | `http://cas.sc.pbc.gov/sso` |
| `serverName` | `WEAVER_SSO_SERVER_NAME` | `http://你的服务地址:8080` |
| `appid` | `WEAVER_SSO_APP_ID` | 管理端注册分配值 |
| `excludePath` | `WeaverSsoFilterConfig` 中 `excludePath` 初始化参数 | 代码内配置 |

## 4. 必填配置

建议使用 `.env` 文件或系统环境变量注入。

### 4.1 基础配置

```properties
SERVER_PORT=8080
SPRING_APPLICATION_NAME=fastgpt-sso-java

SSO_PROVIDER=weaver
SPRING_PROFILES_ACTIVE=weaver
AUTH_TOKEN=请替换为你自己生成的Bearer Token
```

说明：

- `AUTH_TOKEN` 是 FastGPT 调用本服务时使用的 Bearer Token
- FastGPT 管理端里配置的 Token 必须与这里完全一致

### 4.2 泛微 CAS 配置

根据你提供的环境，推荐配置如下：

```properties
WEAVER_SSO_ENABLED=true
WEAVER_SSO_SERVER_URL=http://cas.sc.pbc.gov
WEAVER_SSO_LOGIN_PATH=/sso/login
WEAVER_SSO_APP_ID=管理端注册分配的appid
WEAVER_SSO_SERVER_NAME=http://你的fastgpt-sso-java服务地址:8080
WEAVER_SSO_CALLBACK_PATH=/login/oauth/callback
```

说明：

- `WEAVER_SSO_SERVER_URL` 只填统一认证中心根地址，不带查询串
- `WEAVER_SSO_LOGIN_PATH` 需要按你们现场实际地址填写
- 如果实际登录地址确实是 `http://cas.sc.pbc.gov/sso/login?appid=xxx`，那这里就必须是 `/sso/login`
- 不要继续使用默认的 `/login/login.jsp`，除非你们现场实际就是这个路径
- `WEAVER_SSO_SERVER_NAME` 必须是泛微 CAS 能访问到的地址，不能写前端地址，也不能写本机 `localhost`
- CAS 回调最终会落到：
  `WEAVER_SSO_SERVER_NAME + WEAVER_SSO_CALLBACK_PATH`

最终生成的 CAS 登录 URL 形如：

```text
http://cas.sc.pbc.gov/sso/login?appid=你的appid&service=http%3A%2F%2F你的服务地址%3A8080%2Flogin%2Foauth%2Fcallback
```

### 4.3 CAS 属性映射

```properties
WEAVER_SSO_ATTR_USERNAME=loginid
WEAVER_SSO_ATTR_MEMBER_NAME=lastname
WEAVER_SSO_ATTR_EMAIL=email
WEAVER_SSO_ATTR_MOBILE=mobile
```

说明：

- 这几个字段名必须和泛微 CAS Assertion 实际返回属性名一致
- 如果联调时发现用户名、姓名、邮箱为空，需要打印 `principal.getAttributes()` 后再修正这里
- 当前项目登录名实际使用的是 `principal.getName()`，不是 `attribute.username`
- `attribute.username` 目前主要用于映射和后续扩展，核心登录名仍以 `principal.getName()` 为准

### 4.4 HR 同步接口配置

如果需要使用泛微推送组织和人员数据：

```properties
WEAVER_SSO_HR_SYNC_TOKEN=你自定义的HR同步令牌
```

该值需要与泛微侧调用 `/sso/getdata/**` 接口时使用的鉴权值保持一致。

## 5. FastGPT 管理端如何配置

在 FastGPT 管理端配置 SSO 时，需要把本服务作为标准 SSO 服务接入。

需要关注的值：

- SSO 服务地址：填本服务对外地址，例如 `http://你的服务地址:8080`
- Bearer Token：填 `AUTH_TOKEN`

FastGPT 会调用以下接口：

1. `GET /login/oauth/getAuthURL`
2. `GET /login/oauth/getUserInfo`
3. `GET /user/list`
4. `GET /org/list`

## 8. 联调顺序

建议按下面顺序联调：

1. 在泛微统一认证中心管理端注册本服务，拿到 `appid`
2. 确认泛微允许的回调地址为：
   `http://你的服务地址:8080/login/oauth/callback`
3. 配置本服务环境变量并启动
4. 手工请求：
   `GET /login/oauth/getAuthURL?redirect_uri=FastGPT前端回调地址&state=自定义状态值`
5. 请求头带：
   `Authorization: Bearer 你的AUTH_TOKEN`
6. 检查返回的 `authURL` 是否符合预期
7. 浏览器打开 `authURL`，完成泛微登录
8. 登录成功后，确认是否跳回 FastGPT
9. 检查 FastGPT 是否能继续调用 `GET /login/oauth/getUserInfo`
10. 检查 `/user/list` 和 `/org/list` 返回的数据结构是否满足 FastGPT 导入要求

## 9. 常见问题

### 9.1 回调时报 “Session中未找到CAS Assertion”

通常说明以下问题之一：

- `weaversso.jar` 的 Filter 没有正确注册
- CAS 回调没有经过 `AuthenticationFilter` 和 `SSO20ProxyReceivingTicketValidationFilter`
- `serverName` 填错，导致 Ticket 校验失败
- 浏览器回调时 Session 丢失

排查位置：

- [WeaverSsoFilterConfig.java](/Users/xiaohei666/Documents/FastGPT-dev/fastgpt-sso-java/src/main/java/org/example/config/WeaverSsoFilterConfig.java:65)
- [WeaverSsoProviderServiceImpl.java](/Users/xiaohei666/Documents/FastGPT-dev/fastgpt-sso-java/src/main/java/org/example/service/impl/WeaverSsoProviderServiceImpl.java:82)

### 9.2 能登录，但拿不到姓名或邮箱

说明 CAS 登录成功了，但属性映射名不对。  
需要检查 `principal.getAttributes()` 实际返回的 key，再调整：

- `WEAVER_SSO_ATTR_MEMBER_NAME`
- `WEAVER_SSO_ATTR_EMAIL`
- `WEAVER_SSO_ATTR_MOBILE`

### 9.3 FastGPT 提示接口未授权

通常是下面两个值不一致：

- 本服务 `AUTH_TOKEN`
- FastGPT 管理端里填写的 Bearer Token

### 9.4 FastGPT 跳转地址异常

重点检查：

- `redirect_uri` 是否由 FastGPT 正确传入
- `WEAVER_SSO_SERVER_NAME` 是否为本服务对外地址
- 反向代理是否改写了协议、域名或端口
